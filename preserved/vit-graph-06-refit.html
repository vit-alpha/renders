vit refit — the graph-maintenance loop

Per-tool graph: how edge coefficients learn from observed data. The one per-tool graph with a write path.

Reading the graph. The read loop runs left → right: exercise_relationships.yaml supplies the edges, the library supplies per-exercise gates (measurement tier, form-limited), and workouts supply the observation window. For each surviving edge, curve() fits both ends’ 1RMs, laterality normalisation makes them bilateral-equivalent, and the observed coefficient is compared against the prior. Every edge lands in exactly one of the five proposal classes; all five flow into the dry-run report.

The write path is the dashed right → left track along the bottom: only the drift — proposed update class (heavy outline) passes the --commit gate. apply_refit() then rewrites the yaml textually — coefficient to the observed value, confidence to user-observed — leaving prior_coefficient lines untouched and keeping exactly one trailing newline. Dry-run is the default; --commit opts in (the same convention as vit lib merge).
