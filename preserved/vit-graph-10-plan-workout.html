10 · /plan-workout — skill flow (consumer)

How the planning skill consumes vit data. One of two graphs where the repo is a participant box, not the bounding world.

Reading the graph. Main flow is left → right, consistent with the rest of the series: repo data feeds the vit read queries, the queries feed /plan-workout, and the skill produces a proposed plan in conversation — it never writes files itself. Markdown program docs skip the CLI: the skill reads them directly (curved grey arrow), pulling program context via /program.

This graph (and graph 11) treats the repo as a participant box — a solid-edged actor on the left — rather than the dashed bounding area used everywhere else in the series, because here the interesting structure is between the repo and the skills.

Dashed purple arrows are write-backs: /plan-workout only appends a usage entry to tools/vit/usage_log.md; the actual plan lands on disk via the /layout-plan handoff (plan markdown file, vit wo new record scaffold, activity-log Upcoming entry).
