vit — the tool graph series

Visual maps of how vit's data, tools, and outputs relate. Orientation is constant: data left, tools middle, insights right; time runs down; reversed flow marks a write path.

Key

  • 01 — overview — the most zoomed-out view; acts as the legend for everything below.

The territory

  • 02 — primary — every command, fanned into four families; data split by file type; write paths dashed and reversed.

Per-tool composition

  • 03 — review — three mode-lanes over the shared query layer; the most-composed command.
  • 04 — explain — the auditability command's two lanes; both route anchors through curve().
  • 05 — cluster_progress — how a muscle cluster earns a strength Δ: direct wins, projected median fills, ×1.95 normalises.
  • 06 — refit — the graph-maintenance loop; five proposal classes, one gated write path.

Output construction

  • 07 — the review Δ cell — one number traced from set rows to the table.
  • 08 — the suggest ± band — a distance-plus-age heuristic, drawn as the code computes it.
  • 09 — the summary volume line — load points, the cardio fallback chain, pattern names.

Skill flows

  • 10 — /plan-workout — the consumer: what the planning skill actually reads.
  • 11 — /log-workout — the producer, deliberately drawn right→left: the write path into the repo.
