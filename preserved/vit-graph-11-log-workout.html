11 · /log-workout — skill flow (producer)

How a completed workout gets written into the repo. This graph flows right → left — the deliberate reversal that marks a write path.

Reading the graph — the flow runs right → left, on purpose. Every other graph in the series reads left → right (data → tools → outputs). This one is reversed because it is a write path: sources enter on the right, get parsed and turned into write commands in the middle, and land in repo files on the left. The reversal is the visual signal that arrows here produce data rather than consume it.

Dashed purple arrows are writes. The .wo.yaml record is written entirely via the vit CLI — wo new scaffolds, wo set lands sets (compact notation, incl. --set-note and --machine), wo observation and wo meta carry the workout-level prose, and lib add creates missing library entries; hand-editing survives only for structural reshaping (re-ordering sections, deleting entries). The six lower destinations are direct markdown edits — the staircase fan from the skill's steps 4–5. Solid arrows are handoffs: the three input routes into the skill, and the post-write chain to validation (vit show, vit wo check, vit lib check re-read what was written) and the /update-timeline handoff, whose own write lands in framework/timeline.md.

The repo is a participant box here (as in graph 10), not the dashed bounding area of the other graphs — the point of this page is what crosses the repo boundary, so the repo becomes an actor receiving writes.
